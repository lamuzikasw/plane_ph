# Plane Custom Changelog

Этот файл фиксирует кастомные изменения в Plane, чтобы не терять контекст между длинными чатами, деплоями и новыми задачами.

## Рабочие правила

- Production: https://plane.myneogroup.space/
- Основная ветка кастомного репозитория: `main`
- GitHub remote: `upstream` -> `https://github.com/lamuzikasw/plane_ph.git`
- VPS: `vps.core`
- Деплой делать аккуратно: не трогать БД, API и worker без явной необходимости.
- Для frontend-изменений достаточно собрать новый Docker-образ `web` и переключить только контейнер `web`.
- Перед деплоем проверять хотя бы `pnpm --filter=web check:types` для frontend-правок.
- После деплоя проверять `docker compose ps web` и HTTP `200` на домене.

Закрыты критичные точки аудита: перенос задач между проектами выполняется атомарно, межпроектные связи проверяют доступ к обеим задачам, массовое изменение сроков сохраняет время и не допускает частичного обновления, исправлены периоды и блокеры в аналитике. Из API-логов убраны токены и содержимое запросов, а production-конфигурация теперь требует безопасный `SECRET_KEY`, явный HTTPS CORS allowlist и защищённые cookies.

## P1-аудит: роли и управленческий контекст

Личный дайджест «Сегодня» показывает только задачи текущего сотрудника. Профили команды и управленческая аналитика доступны только роли `OG`, причём ограничение действует одновременно в интерфейсе и API. Операции с ролями принимают только известные значения, не позволяют менять посторонние поля и не предлагают `OG` в обычном onboarding.

Игорь распознаёт 100 вариантов запроса командного недельного отчёта, сохраняет командный контекст в уточняющих сообщениях и подписывает результаты именами исполнителей. Командный отчёт считает только уникальные задачи, назначенные активным сотрудникам, и исключает ботов и задачи без исполнителя.

## P2-аудит: надёжность аналитики

Управленческая аналитика валидирует периоды, фильтры и настройки до выполнения запросов. Пользовательский диапазон ограничен 366 днями; неизвестные разделы, метрики и некорректные UUID/приоритеты возвращают HTTP 400 вместо ошибки сервера или тихо неверного ответа. Настройки защищены схемой и безопасными диапазонами, а CSV-экспорт экранирует значения, которые табличные редакторы могут выполнить как формулы.

## Игорь: разбор больших ТЗ

Игорь принимает технические задания и заметки встречи объёмом до 80 000 символов и 1 200 смысловых пунктов. Большие документы обрабатываются фоновым Celery-worker: текст делится на перекрывающиеся пакеты, результат каждого пакета сохраняется в Redis и затем объединяется. Окно Игоря можно закрыть или перезагрузить страницу — интерфейс восстановит прогресс и готовый разбор.

При временной ошибке повторяется только текущий пакет; уже обработанные части не отправляются в LLM заново. После трёх неудачных попыток Игорь сохраняет частичный прогресс и предлагает вручную перезапустить только проблемные пакеты. Late acknowledgement возвращает работу в очередь при аварийной остановке worker. Распределённые блокировки не дают двум запросам или worker-процессам одновременно обработать одно ТЗ и дважды потратить LLM-запросы.

Для каждого поручения Игорь предлагает самостоятельную задачу с названием, объяснением цели, описанием работ, критериями готовности и открытыми вопросами. Неизвестные цель, срок, приоритет или критерии не придумываются, а явно помечаются для уточнения. Перед созданием сотрудник может отредактировать результат, выбрать проект и исполнителя и снять лишние задачи.

Каждый исходный пункт сохраняется в категории и связывается с предложенными задачами, поэтому интерфейс показывает покрытие поручений. Повторы между пакетами и похожие открытые задачи в выбранном проекте выявляются до записи. Создание остаётся подтверждаемым, атомарным и идемпотентным; в Plane сохраняется структурированное описание с разделами «Зачем», «Что нужно сделать», «Критерии готовности», «Нужно уточнить» и ссылкой на исходные формулировки ТЗ.

## Патч 1.1

Патч 1.1 добавил ИИ-ассистента Игоря, межпроектные связи задач, архив обновлений и оптимизацию загрузки Plane.

### ИИ-ассистент Игорь

В рабочей области появился Игорь — помощник по задачам, срокам и сотрудникам. Он понимает обычные вопросы, находит активные, завершенные, просроченные, заблокированные задачи и задачи без исполнителя, учитывает проект, сотрудника и период, а также сохраняет контекст уточняющих вопросов.

Ответы сопровождаются карточками задач со статусом, сроком, исполнителем и прямой ссылкой. Игорь также поддерживает обычный диалог, если вопрос не относится к работе.

### Межпроектные связи

Задачи можно связывать с задачами из других проектов. Поиск работает по всему рабочему пространству, поддерживает фильтр по проекту и показывает проект в результатах. Исправлено открытие и удаление межпроектных связей.

### Архив обновлений

Раздел "Что нового?" теперь хранит несколько релизов. Патчи 1.1 и 1.0 имеют отдельные URL и переключатель версии. Последний релиз открывается по умолчанию.

Рядом с пунктом "Что нового?" отображается синий индикатор непрочитанного патча. Он исчезает после просмотра последнего релиза и автоматически появляется снова, когда меняется версия `LATEST_RELEASE`.

### Скорость и исправления

Добавлены gzip-сжатие, долговременное кеширование версионированных ассетов, перепроверка HTML и ленивая загрузка `store-context`. Стартовая передача сократилась примерно с 2,41 МБ до 498 КБ. Также исправлено обновление Kanban-доски после переноса задачи в другой проект.

После публикации патча ускорено открытие общего Timeline: ссылка сразу выбирает диаграмму Ганта, а первая страница задач больше не загружается повторно внутри Gantt-компонента. Текст о межпроектных связях переписан на понятных примерах, а карточка производительности сочетает пользовательский результат с техническими деталями и точной метрикой. Заголовки и тесты больше не ограничивают релиз четырьмя карточками и тремя пунктами в каждой. На странице «Что нового?» добавлена нижняя безопасная зона, чтобы плавающая кнопка Игоря не перекрывала навигацию между патчами.

## Патч 1.0

Патч 1.0 превратил Plane из базового таск-трекера в более управленческий инструмент для команды: добавлены аналитика, Gantt, работа с зависимостями, перенос задач между проектами, быстрые действия в Kanban и личная страница "Сегодня".

### 6. Диаграмма Ганта

Добавлена полноценная Timeline/Gantt-визуализация задач. Теперь задачи можно смотреть не только списком или Kanban-доской, но и как календарный план: когда начинается работа, когда заканчивается, какие задачи идут параллельно и где есть пересечения по срокам.

Это помогает руководителю быстро понять, что команда делает сейчас, где сроки наложились друг на друга, какие задачи требуют внимания и как выглядит план по проекту или по всем проектам сразу.

### 7. Связи и зависимости в Ганте

В Gantt добавлено отображение связей между задачами. Задачи можно связывать прямо на диаграмме, видеть стрелки зависимостей и понимать, какая работа блокирует другую.

Также добавлено удаление связи прямо из Gantt через аккуратное меню на русском языке. Это важно, потому что связь легко создать ошибочно, и ее должно быть так же легко убрать без перехода в карточку задачи.

### 8. Улучшенный внешний вид Gantt

Диаграмма стала аккуратнее визуально: задачи, ромбы, линии зависимостей и элементы управления приведены ближе к стилю Plane. Добавлена возможность масштабировать Timeline, чтобы текст задач лучше помещался и план было удобнее читать.

Это делает Gantt не просто технической функцией, а нормальным рабочим экраном, которым можно пользоваться каждый день.

### 9. Перенос задачи между проектами из карточки

Добавлена возможность перемещать задачу из одного проекта в другой прямо из карточки задачи. В модальном окне можно выбрать проект и состояние, после чего задача переносится без ручного пересоздания.

Функция сделана в стиле Asana: пользователь остается в текущем контексте и быстро меняет принадлежность задачи, если она была создана не в том проекте или должна перейти в другую команду.

### 10. Архивация Done одной кнопкой

Добавлено быстрое действие для Kanban: можно архивировать все задачи из столбца Done одной кнопкой.

Это помогает держать доску чистой. Завершенные задачи не мешают текущей работе, но остаются в архиве и не теряются из истории.

### 11. Управленческая аналитика

Добавлена новая аналитика по рабочему пространству: обзор, команда, проекты, загрузка, сроки, риски, качество данных, настройки и задачи.

В аналитике появились карточки с ключевыми показателями: активные сотрудники, активные проекты, задачи в работе, задачи на проверке, просроченные задачи, задачи без исполнителя, задачи без срока, качество данных, риски и другие показатели.

Карточки стали кликабельными там, где это логически нужно. Например, из "Просроченные задачи" можно перейти в список конкретных задач с дедлайнами и ответственными, а из "Без исполнителя" увидеть задачи, которые нужно распределить.

Также добавлены настройки видимости метрик: пользователь может включать и выключать показатели в аналитике без разработки и без изменения кода.

### 12. Подсказки к метрикам

У метрик появились подсказки с иконкой вопроса в кружке. При нажатии открывается короткое объяснение: что считает метрика, откуда берутся данные и как это помогает в работе.

Это нужно для новых сотрудников и руководителей, которые впервые открывают аналитику. Они могут понять смысл показателя прямо внутри Plane, не спрашивая отдельно у команды.

### Дополнительно: страница "Сегодня"

Добавлена личная страница `Сегодня`, которая показывает фокус пользователя на день: задачи на сегодня, риски, блокеры, ближайшие задачи, профиль сотрудника и Daily digest.

Daily digest — это короткая сводка по текущим задачам, срокам и блокерам. Он не отправляется сам по расписанию, а пересобирается из текущих данных: если меняются задачи, сроки, статусы или блокеры, после обновления данных меняется и digest.

Блок "Сделать сегодня" показывает задачи с дедлайном на текущий день. Блок "Ближайшее" показывает назначенные и незакрепленные задачи после сегодняшнего дня, чтобы будущие дедлайны не пропадали.
